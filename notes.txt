Batch Size
    How many alerts you send through the embedding model at once instead of one by one.
    If you embed alerts one at a time, the model does this repeatedly:
        Load alert text
        Tokenize
        Run the neural network forward pass
        Return one vector
    If you have 50,000 alerts, that’s 50,000 forward passes.

The code avoids embedding alerts that cannot form incidents 
A lot of those alerts are “singletons” (candidate groups smaller than min_samples) and can never become a cluster. Embedding them is wasted time.

Why this works:
DBSCAN requires at least min_samples points.
Candidate groups smaller than that will never produce clusters.
Removing them before embeddings can drop workload massively on datasets with many sparse sources.

What to tune for hierarchical
    --dist-threshold plays a similar role to DBSCAN --eps:
        lower = stricter, more clusters
        higher = looser, fewer clusters

    Start with 0.20, 0.25, 0.30 and compare:
        incident count
        mixed-cluster rate (your coherence metric)

Why same-color dots are NOT always physically together
This part is very important.
The short answer:
The map is showing semantic similarity between alerts, not “incident membership geometry.”
The color = incident label
The position = embedding projection
Those are two different things.
So same-color points can appear in multiple places.






Beginner-friendly explanation 

Each color is one attack story.
The dots do not need to touch physically because the map shows alert similarity at the alert level.
If the same attack evolves in phases, targets different hosts, or changes techniques, it may appear as several nearby groups with the same color.
The shared color tells us the engine stitched them into one incident.

That makes the spread-out points feel intentional instead of confusing.
Why this is actually better than forcing one cluster
Forcing same-color points into one geometric blob would actually make the visualization worse.
Current map teaches:

same color = same incident
separate islands = different phases / hosts / stages

That is how real attacks behave.
